# Team Task Manager

## Overview

A full-stack team task manager with authentication, role-based access (Admin / Member), projects, and tasks with due dates. Built on Replit's pnpm monorepo.

## Stack

- **Frontend**: React + Vite + Tailwind v4 + shadcn/ui + wouter + TanStack Query (`artifacts/task-manager`)
- **Backend**: Express 5 + Drizzle ORM (`artifacts/api-server`)
- **Database**: Replit-managed PostgreSQL (Drizzle ORM is used in place of Mongoose; same data modeling, but uses Postgres so no external DB hosting is required)
- **Auth**: JWT (jsonwebtoken) + bcryptjs password hashing
- **Validation**: Zod schemas generated from `lib/api-spec/openapi.yaml`
- **API client**: TanStack Query hooks generated by Orval

## Folder structure

```

  api-server/          Express API (JWT auth, projects, tasks, dashboard)
    src/
      lib/auth.ts             JWT signing + bcrypt helpers
      middlewares/auth.ts     requireAuth + requireAdmin middleware
      middlewares/errorHandler.ts
      routes/auth.ts          /api/auth/signup|login|me
      routes/users.ts         /api/users
      routes/projects.ts      /api/projects (admin-gated mutations)
      routes/tasks.ts         /api/projects/:id/tasks, /api/tasks/:id, /api/tasks/me
      routes/dashboard.ts     /api/dashboard/summary, /api/dashboard/recent
  task-manager/        React + Vite frontend (login, dashboard, projects, my-tasks)

  api-spec/            OpenAPI source of truth (openapi.yaml)
  api-zod/             Generated Zod schemas (server validation)
  api-client-react/    Generated TanStack Query hooks (frontend)
  db/                  Drizzle schema + connection (users, projects, project_members, tasks)
```

## Key Commands

- `pnpm run typecheck` ‚Äî full typecheck across all packages
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API hooks + Zod schemas after editing `openapi.yaml`
- `pnpm --filter @workspace/db run push` ‚Äî push DB schema changes (dev only)

## Run locally

The Replit workspace runs the API and frontend automatically via workflows. Open the preview pane to see the app at `/`.

If running outside Replit:

```bash
pnpm install
# set DATABASE_URL and SESSION_SECRET (or JWT_SECRET) in your env
pnpm --filter @workspace/db run push
PORT=8080 pnpm --filter @workspace/api-server run dev    # API on :8080
PORT=5173 BASE_PATH=/ pnpm --filter @workspace/task-manager run dev  # web
```

The frontend talks to `/api/...` and assumes it's served behind the same origin as the API (or proxied to it).

## Deploying

### On Replit (recommended)
1. Click **Publish** in the Replit workspace.
2. Replit handles building, hosting, TLS, and health checks.
3. Make sure `DATABASE_URL` and `SESSION_SECRET` are set as deployment secrets ‚Äî Replit's PostgreSQL provisioning handles `DATABASE_URL` automatically.

### On Railway (manual)
1. Push this repo to GitHub.
2. Create two Railway services backed by this repo:
   - **API** ‚Äî Build: `pnpm install && pnpm --filter @workspace/api-server run build`, Start: `pnpm --filter @workspace/api-server run start`. Provide `PORT`, `DATABASE_URL` (from Railway's Postgres plugin), and `JWT_SECRET`.
   - **Web** ‚Äî Build: `pnpm install && BASE_PATH=/ pnpm --filter @workspace/task-manager run build`, Serve the `artifacts/task-manager/dist/public` folder as static (use a static-site service or `pnpm --filter @workspace/task-manager run serve` with `PORT`).
3. Run `pnpm --filter @workspace/db run push` once against the Railway Postgres URL to create the schema.
4. Point a public domain or proxy so `/api/*` requests reach the API service and everything else reaches the web service.

### Environment variables

| Variable | Where | Purpose |
| --- | --- | --- |
| `DATABASE_URL` | API server | Postgres connection string |
| `SESSION_SECRET` or `JWT_SECRET` | API server | Secret used to sign JWTs |
| `PORT` | both services | Port the dev/preview server binds to |
| `BASE_PATH` | web (build/dev) | URL prefix the SPA is served from (`/`) |
